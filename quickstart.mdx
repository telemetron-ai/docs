## Overview

Sync customer and device data to enable Telemetron's AI-powered support. Complete setup in minutes.

**Goal**: Create device-to-customer mappings so telemetry reaches the right customers.

### Step 1: Get Your API Key

<AccordionGroup>
  <Accordion icon="key" title="Generate an API Key">
    1. Log into your Telemetron dashboard at [admin.telemetron.ai](https://admin.telemetron.ai)
    2. Navigate to **Settings** > **API Keys**
    3. Click **Generate New API Key**
    4. Copy and securely store your API key
    
    <Warning>Keep your API key secure! Never expose it in client-side code or commit it to version control.</Warning>
  </Accordion>
  
  <Accordion icon="shield-check" title="Authentication">
    All API requests require your API key in the request headers:
    
    ```bash
    x-api-key: your_api_key_here
    ```
    
    The API key is scoped to your organization, so all operations will be isolated to your organization's data.
  </Accordion>
</AccordionGroup>

### Step 2: Create Customer Record

<AccordionGroup>
  <Accordion icon="user-plus" title="Create or update customer">
    Sync customer information to Telemetron:

    <CodeGroup>
    ```bash cURL
    curl -X POST https://admin.telemetron.ai/api/ext-v1/customer/createOrUpdateCustomer \
      -H "x-api-key: your_api_key_here" \
      -H "Content-Type: application/json" \
      -d '{
        "name": "John Doe",
        "email": "john.doe@example.com",
        "phone": "+1234567890",
        "address": "123 Main St, City, State 12345"
      }'
    ```

    ```javascript JavaScript
    const response = await fetch(
      'https://admin.telemetron.ai/api/ext-v1/customer/createOrUpdateCustomer',
      {
        method: 'POST',
        headers: {
          'x-api-key': 'your_api_key_here',
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({
          name: 'John Doe',
          email: 'john.doe@example.com',
          phone: '+1234567890',
          address: '123 Main St, City, State 12345'
        })
      }
    );
    
    const customer = await response.json();
    console.log('Customer ID:', customer.telemetronCustomerId);
    ```

    ```python Python
    import requests

    response = requests.post(
        'https://admin.telemetron.ai/api/ext-v1/customer/createOrUpdateCustomer',
        headers={
            'x-api-key': 'your_api_key_here',
            'Content-Type': 'application/json'
        },
        json={
            'name': 'John Doe',
            'email': 'john.doe@example.com',
            'phone': '+1234567890',
            'address': '123 Main St, City, State 12345'
        }
    )
    
    customer = response.json()
    print('Customer ID:', customer['telemetronCustomerId'])
    ```
    </CodeGroup>

    <Tip>Save the `telemetronCustomerId` from the response - you'll need it for device assignments!</Tip>
  </Accordion>

  <Accordion icon="magnifying-glass" title="Check if customer exists">
    Query by email to check existence:

    ```bash
    curl -X POST https://admin.telemetron.ai/api/ext-v1/customer/queryCustomer \
      -H "x-api-key: your_api_key_here" \
      -H "Content-Type: application/json" \
      -d '{"email": "john.doe@example.com"}'
    ```

    Returns full customer record including assigned devices.
  </Accordion>
</AccordionGroup>

### Step 3: Map Devices to Customers

<AccordionGroup>
  <Accordion icon="microchip" title="Assign device to customer">
    Create the device-to-customer mapping for telemetry routing:

    <CodeGroup>
    ```bash cURL
    curl -X POST https://admin.telemetron.ai/api/ext-v1/deviceAssignment/assignDeviceToOwners \
      -H "x-api-key: your_api_key_here" \
      -H "Content-Type: application/json" \
      -d '{
        "deviceIdentifier": "SN-12345-ABCDE",
        "customerEmails": ["john.doe@example.com"],
        "autoCreateDevice": true,
        "deviceType": "Thermostat"
      }'
    ```

    ```javascript JavaScript
    const response = await fetch(
      'https://admin.telemetron.ai/api/ext-v1/deviceAssignment/assignDeviceToOwners',
      {
        method: 'POST',
        headers: {
          'x-api-key': 'your_api_key_here',
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({
          deviceIdentifier: 'SN-12345-ABCDE',
          customerEmails: ['john.doe@example.com'],
          autoCreateDevice: true,
          deviceType: 'Thermostat'
        })
      }
    );
    
    const result = await response.json();
    console.log('Device assigned:', result);
    ```

    ```python Python
    import requests

    response = requests.post(
        'https://admin.telemetron.ai/api/ext-v1/deviceAssignment/assignDeviceToOwners',
        headers={
            'x-api-key': 'your_api_key_here',
            'Content-Type': 'application/json'
        },
        json={
            'deviceIdentifier': 'SN-12345-ABCDE',
            'customerEmails': ['john.doe@example.com'],
            'autoCreateDevice': True,
            'deviceType': 'Thermostat'
        }
    )
    
    result = response.json()
    print('Device assigned:', result)
    ```
    </CodeGroup>

    <Info>
      Set `autoCreateDevice: true` to automatically create the device if it doesn't exist. When using this option, you **must** provide `deviceType`. You can assign a device to multiple customers by adding more emails to the `customerEmails` array.
    </Info>
  </Accordion>
</AccordionGroup>

<Tip>
  **Result**: Telemetry from `SN-12345-ABCDE` now routes to `john.doe@example.com` for AI-powered support.
</Tip>

## Integration Patterns

  <Card title="Product Registration" icon="clipboard-list">
    1. Query/create customer
    2. Assign device (`autoCreateDevice: true`)
    3. ✅ Telemetry linked
  </Card>

  <Card title="Device Transfer" icon="right-left">
    1. Unassign from old owner
    2. Assign to new owner
    3. ✅ Telemetry rerouted
  </Card>

    3. ✅ All owners get alerts

## Best Practices

| Category | Recommendations |
|----------|-----------------|
| **API Keys** | Store in env vars • Use separate keys for dev/prod • Rotate regularly • Never commit to code |
| **Error Handling** | Check status codes • Handle 404/409/500 appropriately • Implement retry logic |
| **Device IDs** | Use consistent identifiers • Ensure uniqueness • Standardize format |
| **Performance** | Batch operations • Cache when appropriate • Implement rate limiting |
